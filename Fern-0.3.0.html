Added

  • Spell check. Misspelled words are underlined in the editor with corrections on tap/right-click. On by default — turn it off in Settings → General → Editing.
  • The / quick-insert menu now works on iPad, matching Mac: type / at the start of a line for headings, lists, quotes, and links — tap a row (or arrow down and Return with a hardware keyboard) to insert it. Sized compact for iPad's smaller screen.

Changed

  • Atlas is now Fern. New name, same app — too many markdown apps were called Atlas. Nothing about your notes changes: your vault, folders, Regions, and Trails are exactly where they were. Fern installs as a new app, so point it at your existing vault and re-enter your API keys — the index rebuilds on its own.
  • Return on an indented list item now outdents one level and keeps the list style, instead of leaving the list entirely. Press Return again once you're back at the base level to drop out of the list.
  • More room around checklist and bullet markers — the gap between a checkbox or bullet and its text, and the space between list items, both grew for legibility.
  • Trails from ⌘K are no longer capped near 4-5 stops. A topic you've written about extensively can now trace a much longer path; a narrow one still gets a short, focused list.
  • iPad: tapping a command palette row now selects it first and activates on a second tap — the same Raycast-style model Mac already used — instead of acting on the first tap.
  • iPad: the palette's answer, tensions, loops, and trail cards now cap their height to the room actually available in the window instead of a fixed size, so they can't run off the bottom edge on a smaller iPad.
  • iPad: the View menu no longer lists "Show/Hide Sidebar" twice.
  • The iPad context panel is now a dedicated ⓘ button in the toolbar, opening Mac's relationships-first layout (tags, outline, links, backlinks, then Nearby Notes) in a popover sized to its content, rather than a card floating over the note. The note's ••• menu no longer lists a redundant "Note Information" entry there — iPhone (which has no ⓘ button) still has it, and it now toggles the info sheet open and closed instead of only opening it.
  • The panel's word/character/reading-time detail (tap the footer line) now opens as a plain attached flyout — square corners, no speech- bubble arrow — instead of a system popover, so it doesn't nest awkwardly inside iPad's own info popover.

Fixed

  • iPad: tapping the Done checkmark while editing now also closes the Note Information panel.
  • iPad: the toolbar's Liquid Glass pill no longer reserves an empty gap next to the note's ••• menu — the (invisible) Done checkmark button was still occupying a slot in the shared glass background even at 0 opacity.
  • iPad: the Done checkmark no longer stays stuck on-screen after navigating to All Notes while a note was being edited — leaving a note now ends its edit session.
  • iPad: the Done checkmark now fades away in place instead of animating off-screen when it disappears.
  • iPad: typing # in the command palette with a hardware keyboard attached could jump to a recent note and close the palette instead of opening tag search (a Shift-key mismatch in the 1–5 recent-note shortcut).
