Added

  • Exclude any note from AI. Turn on Exclude from AI and that note's content never leaves your device again — no embedding, no Ask answers, no discovered connections. Search, links, and tags keep working as before.
  • Available from a note's ••• menu, its context menu in the sidebar or All Notes, or the context panel.
  • The first time you use it, Atlas explains what stops — and that content already sent can't be recalled.
  • Flip it back on anytime to resume indexing.
  • Pull-to-search and a floating New Note button on iPad, matching iPhone. Pull down from the top of a note or All Notes to open ⌘K, and tap the new floating pencil button to start a note.
  • Requires no hardware keyboard attached. A Done checkmark also appears while editing to dismiss the keyboard.
  • "Trace a Path" is now in the Note menu and ⌘K, not just the context panel — the same reading-path trace, one more way to start it.

Changed

  • The Mac context panel was redesigned. Relationships come first — tags, Region membership, a live outline that highlights the heading you're reading, then links and backlinks. Discovered Connections now live in one intelligence-tinted section below, with a single footer line for word count and last-edited time.
  • Daily notes show their raw date (e.g. "2026-07-08") by default again. Readable dates ("July 8, 2026") are still available — turn them on in Settings → Daily Notes → Readable dates.
  • Tags now work mid-sentence. (#health), "#focus", and #goal are recognized as tags — previously #tag only counted at the start of a line or right after a space.
  • The app opens instantly. Atlas now remembers your notes between sessions, so launching — especially on iPhone and iPad — shows your vault immediately instead of a loading bar while every note is re-read from iCloud.
  • Changes made on other devices while the app was closed still arrive within moments of opening.
  • The first launch after this update loads the old way once; every launch after that is instant.

Fixed

  • iCloud daily-note forks now heal automatically. If two devices created the same day's note before syncing caught up, iCloud used to leave both files behind (e.g. a stray "2026-07-08 2.md"). Atlas now detects the fork and folds it back into the day's real note.
  • Tagging a note now reliably files it into its Region. Two cases used to leave a tagged note missing from its Region: adding the tag via the context panel's tag chip, and typing a #tag right after retitling a note.
  • Removing a tag could likewise leave a note stuck in a Region — also fixed. Tag a note either way and it now appears in the Region's member list immediately.
  • Expanding a Nearby Notes suggestion now dims the whole window — the sidebar and side panels included — instead of only the note, so the detail card reads clearly as the focus.
  • The brief blank screen some launches showed before the loading bar now says "Opening your notes…" so the app never looks stalled.
